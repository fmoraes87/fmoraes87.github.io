Installation

The first step is to install Java SE Development Kit 11.X.Y (JavaSDK). You’ll need an internet connection for the download.

The following steps install the latest stable version of the JavaSDK. Java's stability guarantees ensure that all the examples in the book that compile will continue to compile with newer Java versions. The output might differ slightly between versions, because Java often improves error messages and warnings. In other words, any newer, stable version of Java you install using these steps should work as expected with the content of this book.

Command Line Notation

In this chapter and throughout the book, we’ll show some commands used in the terminal. Lines that you should enter in a terminal all start with $. You don’t need to type in the $ character; it indicates the start of each command. Lines that don’t start with $ typically show the output of the previous command. Additionally, PowerShell-specific examples will use > rather than $.

Troubleshooting

To check whether you have Java installed correctly, open a shell and enter this line:

$ java -version

You should see the java version number as follow.

java version "x.y.z_nnn"
Java(TM) SE Runtime Environment (x.y.z_nnn-ww)
...

If you see this information, you have installed Java SDK successfully! If you don’t see this information and you’re on Windows, check that Java is in your %PATH% system variable. If that’s all correct and Java still isn’t working, there are a number of places you can get help. A great resources is Stack Overflow. More information is available at the Oracle page

Hello, World!

Now that you’ve installed Java, let’s write your first Java program. It’s traditional when learning a new language to write a little program that prints the text Hello, world! to the screen, so we’ll do the same here!

Note: This book assumes basic familiarity with the command line. Java makes no specific demands about your editing or tooling or where your code lives, so if you prefer to use an integrated development environment (IDE) instead of the command line, feel free to use your favorite IDE.

Creating a Project Directory

You’ll start by making a directory to store your Java code. It doesn’t matter to Java where your code lives, but for the exercises and projects in this book, we suggest making a java_projects directory in your home directory and keeping all your projects there.

Open a terminal and enter the following commands to make a java_projects directory and a directory for the Hello, world! project within the java_projects directory.

For Linux, macOS, and PowerShell on Windows, enter this:

$ mkdir ~/java_projects
$ cd ~/java_projects
$ mkdir hello_world
$ cd hello_world

For Windows CMD, enter this:

> mkdir "%USERPROFILE%\java_projects"
> cd /d "%USERPROFILE%\java_projects"
> mkdir hello_world
> cd hello_world

Writing and Running a Java Program

Next, make a new source file and call it HelloWorld.java. Java files always end with the .java extension. If you’re using more than one word in your filename, use CamelCase to indicate them. For example, use MyHome.java rather than myhome.java. Repare that we are using "H" (upper case) instead of "h" (lower case). Class names should be nouns, in mixed case with the first letter of each internal word capitalized. Try to keep your class names simple and descriptive. Use whole words-avoid acronyms and abbreviations (unless the abbreviation is much more widely used than the long form, such as URL or HTML). For more informations please check Java Naming Convention.

Now open the HelloWorld.java file you just created and enter the code in Listing 1-1.

Filename: HelloWorld.java

public class HelloWorld {
	public static void main (String[] args){
		System.out.println("Hello, world!");
	}
}

Listing 1-1: A program that prints Hello, world!

Save the file and go back to your terminal window. On terminal enter the following commands to compile:

$ javac HelloWorld.java

That will create a new file called HelloWorld.class. Afterwards run the command:

$ java HelloWorld.java
Hello, world!

Regardless of your operating system, the string Hello, world! should print to the terminal. If you don’t see this output, refer back to the “Troubleshooting” part of the Installation section for ways to get help.

If Hello, world! did print, congratulations! You’ve officially written a Java program. That makes you a Java programmer—welcome!

Anatomy of a Java Program

Let’s review in detail what just happened in your Hello, world! program. Here’s the first piece of the puzzle:

public class HelloWorld {

...
}

These lines define a java class. A class is declared using the keyword class, followed by the class name. The keyword public is called modifier. You will learn more about it later. Also, note that the class body is wrapped in curly brackets, {}. Java requires these around all function/method bodies. It’s good style to place the opening curly bracket on the same line as the function declaration, adding one space in between.

Java is case sensitive

It does not recognize the following two words as being the same thing: Class and class.

Inside the java class is the following code:

	public static void main (String[] args){
		...
	}

These lines define a method in Java. The main method is special: it is always the first code that runs in every executable Java program. The first line declares a method named main that has an array of strings (String [] args) as parameter and returns nothing (void). Also, note that the function body is wrapped in curly brackets, {}. A class containing a main method is referred as a "main class". Any program that you want to run must have a public main method. We'll learn more about how to declare method/functions later.

Brackets ( [] ) can be placed to the right of String or to the right of args, but the former is recommended: (String [] args) or (String args [])

Inside the main function is the following code:

	System.out.println("Hello, world!");

This line does all the work in this little program: it prints text to the screen.

Going deeper

System is a class in the java.lang package . The out is a static member of the System class, and is an instance of java.io.PrintStream. The println is a method of java.io.PrintStream. This method is overloaded to print message to output destination, which is typically a console or file.

Compiling and Running Are Separate Steps

You’ve just run a newly created program, so let’s examine each step in the process.

Before running a Java program, you must? (we will talk a little bit more about this later in this chapter) compile it using the Java compiler by entering the javac command and passing it the name of your source file, like this:

$ javac HelloWord.java

If you have a C or C++ background, you’ll notice that this is similar to gcc or clang. After compiling successfully, Java outputs a bytecode.

On Linux, macOS, and PowerShell on Windows, you can see the bytecode by entering the ls command in your shell. On Linux and macOS, you’ll see two files. With PowerShell on Windows, you’ll see the same files that you would see using CMD.

$ ls
HelloWord.java  HelloWord.class

With CMD on Windows, you would enter the following:

> dir /B %= the /B option says to only show the file names =%
HelloWord.java 
HelloWord.class

This shows the source code file with the .java extension and the bytecode file .class extension.

If HelloWord.java was your Hello, world! program, this line would print Hello, world! to your terminal.

If you’re more familiar with a dynamic language, such as Python or JavaScript, you might not be used to compiling and running a program as separate steps.

Just compiling with javac is fine for simple programs, but as your project grows, you’ll want to manage all the options and make it easy to share your code. Next, we’ll introduce you to the Maven tool, which will help you write real-world Java programs.

Running a java code without compilation

Since Java 11 you can use the follow command to run a little program without compiling it:

$ java HelloWord.java

The output should be:

Hello, world!

This feature was introduced as a part of JEP 330: Launch Single-File Source-Code Programs.
